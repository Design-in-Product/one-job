## The add-a-card archaeology (Xian asked how the old model got lost)

- **Pre-pivot (tabbed UI)**: the Stack tab rendered TaskForm *persistently
  below the card stack* — an always-visible "Add New Task" button on the
  main screen (old Index.tsx line ~490). It even had an open bug for being
  cut off at the bottom (ADD-BTN-001) — evidence it was a permanent fixture.
- **The 2025-08-06 Card Deck pivot spec** declared "decluttered by default —
  all text/UI chrome removed from viewport." Add Task was demoted to the
  long-press arc menu — and even that handler was left unwired (menu just
  closed) until the 2026-07-01 rebuild.
- **Conclusion**: the persistent affordance was removed *deliberately* by
  the decluttering principle, not lost accidentally — but real use shows
  the principle over-corrected for the single most frequent input action.
  Capture speed beats chrome purity when a thought is evaporating.

## Options prepared for discussion

A. Persistent floating "+" (small coral FAB, bottom-right) → Add Task modal
B. Pull-down on the deck "deals a blank card" (Clear-style, chromeless,
   on-metaphor; less discoverable — candidate to ADD later, not replace)
C. Tap the deck underlay to add (rejected: invisible, misfire-prone)
D. Status quo, arc menu only (rejected by dogfooding)

Recommendation: A now (keep arc menu path too), B later as the power
gesture. Prototype built on branch for Xian's review — not merged.
