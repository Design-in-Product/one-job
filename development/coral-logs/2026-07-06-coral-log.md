## 🔬 Confession with a fix: `npx tsc --noEmit` was checking NOTHING

The root tsconfig has `"files": []` + project references — bare
`tsc --noEmit` type-checked zero files, exit 0, every time. Every
"tsc clean" claim in previous logs was hollow; the real nets were
vitest and the vite build (which transpiles but doesn't type-check).
Discovered when the v2 type rename produced an impossible clean pass.
Correct invocation: **`npx tsc --noEmit -p tsconfig.app.json`** —
which immediately found all 30+ legitimate errors and now guards the
repo. *Cross-pollination lesson: a check that always passes is
indistinguishable from a check that never runs — when a gate has
never once failed, verify the gate itself.* (Third instance of the
observed-outcomes law in one week: deploys, exports, and now the
type-checker.)
