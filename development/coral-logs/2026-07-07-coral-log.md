## Session #3: iPad "completed tasks not in Done" report

Reproduced Xian's exact scenario in Chromium (390x844, seeded "test"
card with 2 unfinished sub-cards + one plain card):
- Plain card swipe-right -> lands in Done room, tab reads "Done (1)". OK
- "test" (badge 2) swipe-right -> completion REFUSED by design (Item
  15): toast "2 sub-tasks still open inside — finish or clear them
  first" + auto-descent into the blocking sub-deck. OK
Diagnosis: not a persistence bug — the screenshots show the blocked
case (badge 2 visible on the card). Done is empty because nothing
completed. Two honesty gaps exposed, reported to Xian for a decision:
(1) mid-drag the card shows the green "Done" pill even when release
will bounce — the affordance promises an outcome the release refuses;
(2) completed SUB-cards never surface in the Done room (chain is
top-level only), so "I completed things and Done is empty" is also
true if the things were sub-tasks.

Tooling lesson: `pkill -f vite` inside a compound command kills the
shell running it (the pattern matches the command line itself) — the
exit-144s earlier in the session were self-inflicted.

**Session #3 addendum — Xian pushed back ("nothing is showing up done
anymore"), and he's right.** Systematic hunt on the PRODUCTION bundle:
- Web persistence exonerated: plain card complete -> Done survives
  reload (an earlier scare was my own harness: addInitScript re-seeds
  on every navigation — success signals, again).
- Native boot exonerated: hydrateFromNativeStorage is awaited before
  render, and the store is constructed lazily on first render.
- THE FINDING: a card completed INSIDE an interior deck disappears from
  every surface — gone from the sub-deck view (which filters to active),
  never shown in the Done room (ChainView filters top-level only). The
  data is safe (completed:true + completedAt persist and survive
  reload) but the work is invisible. Since Xian now lives inside his
  big interior deck, every completion he makes vanishes: "nothing is
  showing up done anymore" is literally the designed behavior composing
  into a trust bug. Fix surface (where completed sub-cards should
  appear) is his design call — recommended flattened Done room with
  parent breadcrumb; awaiting his word.
