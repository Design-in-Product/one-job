## Item 13 — 🔴 updates wipe his local data (release blocker, live)

The single most important bug report the project has had: his deck
does not survive our pushes. Investigated the full web path — nothing
can legally clear `oneJobTasks` on deploy (localStorage is
origin-scoped; SW updates don't touch it; demo reset only clears demo
keys). Found ONE real landmine while looking: on JSON parse failure
the store *reseeded and overwrote* the unreadable-but-recoverable data.

**Shipped (test-first, 6 new tests, 19 total green):** the store
safety net — `.meta` (count/timestamp) on every save, dated
`.snapshot.YYYY-MM-DD` copies (7-day retention, empty-never-overwrites-
non-empty guard), automatic restore of the newest readable snapshot
when the main key vanishes but meta says data existed, and corruption
now quarantined to `.corrupt.<ts>` instead of clobbered. demo.html's
reset updated to clear the safety-net keys too (or reset would
self-defeat). Verified end-to-end in Chromium: create card → delete
main key → reload → card back on deck, restore warning in console.

**Root cause still open** — the safety net protects against key-level
loss, but if his runtime is wiping ALL site data (native reinstall,
iOS Safari-tab eviction, dev-server origin change), only export/mirror
survives. Asked Xian how he runs the app when the wipe hits.

## Innovations worth cross-pollinating

- **In-progress as emergent structure, not status**: the parent card is
  "in progress" exactly when its interior isn't finished — a state
  machine where the state is *readable from the containment tree*
  rather than stored. Products drowning in status enums could steal
  this: most "statuses" are derivable facts.
- **The safety-net pattern for local-first apps**: meta + dated
  snapshots + quarantine-don't-clobber + restore-on-missing, ~60 lines,
  test-first. Any sibling holding user data in localStorage should have
  exactly this layer between "the browser lost it" and "the user lost
  it."
- **Dogfooding as capture channel**: Xian keeps product notes as cards
  in the product — the feedback loop IS the usage loop.
